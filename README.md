# Converter Pro

Онлайн-конвертер единиц (давление, вес, длина, площадь, объём, температура, скорость, время, данные, энергия). Три языка: русский, қазақша, English.

---

## Как публиковать через GitHub

### Шаг 1. Репозиторий на GitHub

1. Создай репозиторий на [github.com](https://github.com) (New repository).
2. В папке проекта выполни (если ещё не делал):
   ```bash
   git init
   git add .
   git commit -m "Converter Pro"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_ЛОГИН/НАЗВАНИЕ_РЕПО.git
   git push -u origin main
   ```

### Шаг 2. Где будет жить сайт

Next.js удобнее всего поднимать на **Vercel** (бесплатно, один клик из GitHub).

1. Зайди на [vercel.com](https://vercel.com), войди через GitHub.
2. **Add New** → **Project** → выбери свой репозиторий.
3. Жми **Deploy**. Vercel сам сделает `npm run build` и задеплоит.
4. После деплоя тебе дадут ссылку вида `https://твой-проект.vercel.app`.

Дальше каждый `git push` в `main` будет автоматически пересобирать и обновлять сайт.

## Переменная окружения (NEXT_PUBLIC_SITE_URL)

**Что это:** полный адрес сайта. Он подставляется в sitemap.xml, в canonical-ссылки и в Open Graph, чтобы поисковики и соцсети видели правильный URL.

**Что делать:**

- **Локально** — можно ничего не делать, в коде уже стоит запасной адрес `https://converter-pro.app`.
- **На Vercel:** после первого деплоя зайди в проект → **Settings** → **Environment Variables** → добавь:
  - **Name:** `NEXT_PUBLIC_SITE_URL`
  - **Value:** твой реальный адрес, например `https://твой-проект.vercel.app` или свой домен.
  Сохрани и сделай **Redeploy** проекта.

Если не задать переменную на проде, в sitemap и в мета-тегах будет подставляться запасной URL из кода.

## Почему при сборке столько страниц (тысячи)

При `npm run build` Next.js генерирует **статические страницы** для всех маршрутов:

- главная и разделы (конвертеры, статьи, гайды, FAQ и т.д.);
- каждая категория (вес, длина, давление и т.п.);
- каждая пара единиц (например bar→psi, kg→lb);
- то же самое для трёх языков (ru, kk, en).

Получается много страниц — это нормально и нужно, чтобы всё было быстрым и удобным для поисковиков. Сборка один раз делается на сервере (например на Vercel), пользователи просто открывают уже готовые страницы.

## Размер билда и деплой на Vercel

- **Локально** папка `.next` может разрастаться до гигабайтов, если часто запускать `npm run dev`: в неё пишется кэш Turbopack (`.next/dev/cache/`). На Vercel `next dev` не запускают, там только `next build`, поэтому такого кэша нет.
- В репозиторий **не попадают** `.next` и `node_modules` (они в `.gitignore`). Vercel клонирует репо и сам выполняет `npm install` и `next build`.
- В проекте включён **`output: 'standalone'`**: билд собирается в урезанный артефакт без лишних файлов, что уменьшает размер деплоя при большом числе статических страниц (у тебя их тысячи — конвертеры × языки × пары).
- Если когда-то закоммитил `.next` или `node_modules`, удали их из истории и снова добавь в `.gitignore`, затем сделай новый коммит без них.

Чтобы оценить размер именно **продакшен-билда** (как на Vercel), локально выполни:
```bash
Remove-Item .next -Recurse -Force -ErrorAction SilentlyContinue
npm run build
```
Потом посмотри размер папки `.next` — это и есть то, что по сути деплоится (на Vercel используется standalone-вариант этого билда).

## Команды

| Команда | Что делает |
|--------|------------|
| `npm install` | Установка зависимостей |
| `npm run dev` | Запуск сайта локально (обычно http://localhost:3000) |
| `npm run build` | Сборка для продакшена |
| `npm run start` | Запуск уже собранного проекта |
| `npm run favicons` | Пересоздать favicon.ico, favicon-96x96.png, apple-touch-icon.png из `public/favicon.svg` |

## Если хочешь именно GitHub Pages

GitHub Pages раздаёт только статику (HTML/CSS/JS), без Node-сервера. Чтобы выложить этот проект туда, нужно:

1. Включить в Next.js статический экспорт (в `next.config.ts` добавить `output: 'export'`).
2. Собирать проект (получится папка `out/`) и заливать её содержимое в ветку `gh-pages` или в отдельную папку (например `docs/`) через GitHub Actions.

Это уже отдельная настройка; проще всего для этого проекта — Vercel + GitHub, как в шагах выше.
